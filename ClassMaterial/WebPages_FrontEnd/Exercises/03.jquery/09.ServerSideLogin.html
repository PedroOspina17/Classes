AJAX

In this exercise you will be applying the knowledge about http requests and ajax.

Using the localstorage result form, change it to use the security backend provided. In the following URL is published the security api ready to use. Security API
this api has the following endpoints:

  • URL: /Healthcheck - Type: GET: to identify if the api is working with get.
  • URL: /LogIn - Type: POST: to validate if the user is register in the application and verify if the password is right.
    POST: http://3.14.144.130/LogIn
    Data: 
    
                    {                
                        "userName": "PedroOspina172",
                        "password": "Admin1234"
                    }
  • URL: /GetUsers - Type: GET: to get all the existing users. returns a list of users that are already created. Use this to see how is the information stored.
    GET: http://3.14.144.130/GetUsers
  • URL: /GetUser - Type: GET: to get an specific user already exists by its username.
    GET: http://3.14.144.130/GetUser?userName=PedroOspina17
  • URL: /CreateUser - Type: POST: to register a new user.
    POST: http://3.14.144.130/CreateUser
    Data: 
    
                {
                    "age": 3, 
                    "lastName": "Ospina 2", 
                    "name": "Pedro 2", 
                    "password": "Admin1234", 
                    "userName": "PedroOspina172"
                }
  • URL: /DeleteUser - Type: POST: to delete an existing user.
    POST: http://3.14.144.130/DeleteUser
    Data: 
    
                    {
                        "userName": "PedroOspina172"
                    }
